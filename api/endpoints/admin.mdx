---
title: "Admin"
description: "Access session summaries and export full conversation data for expert review and audit."
---

The admin endpoints provide access to persisted session data stored in PostgreSQL. Unlike the session endpoints, which operate on in-memory state, these endpoints query the database directly and remain available after server restarts.

<Warning>
  These endpoints are intended for internal use by academic directors and program coordinators. Restrict access to authenticated users in production deployments.
</Warning>

---

## GET /api/admin/sessions/summary

Returns a summary of all evaluation sessions stored in the database, suitable for display in an expert review dashboard.

### Response

<ResponseField name="total_sessions" type="number" required>
  Total number of sessions in the database.
</ResponseField>

<ResponseField name="sessions" type="object[]" required>
  Array of session summary objects from the database. Each object contains the session's status, completion percentage, and key profile metadata.
</ResponseField>

<ResponseField name="export_timestamp" type="string" required>
  ISO 8601 timestamp of when the summary was generated.
</ResponseField>

### Example

```bash
curl --request GET \
  --url http://localhost:8000/api/admin/sessions/summary
```

```json
{
  "total_sessions": 3,
  "sessions": [
    {

      "user_profile": {
        "initial_message": "Hola, me gustaría comenzar"
      }
    },
    {
      "session_id": "7b1a3c9e-2d41-4e88-b17f-5a8c9d0e1f23",
```

## GET /api/admin/export/{session_id}

Exports the complete data for a single session, including the full conversation transcript, agent evaluations per competency, and an evaluation summary. This export is the primary input for human expert review.

### Path parameters

<ParamField path="session_id" type="string" required>
  The UUID of the session to export.
</ParamField>

### Response

<ResponseField name="session_id" type="string" required>
  The session identifier from the request.
</ResponseField>

<ResponseField name="export_timestamp" type="string" required>
  ISO 8601 timestamp of when this export was generated.
</ResponseField>

<ResponseField name="user_profile" type="object" required>
  The user profile object as stored in the database at session creation. Contains at minimum `initial_message` and any additional profile data collected during the assessment.
</ResponseField>

<ResponseField name="conversation_summary" type="object" required>
  High-level statistics about the conversation.

  <Expandable title="conversation_summary properties">
    <ResponseField name="status" type="string" required>
      Session status: `active`, `completed`, or `abandoned`.
    </ResponseField>
    <ResponseField name="duration_minutes" type="number | null" required>
      Duration of the session in minutes. `null` if the session has not ended or timestamps are unavailable.
    </ResponseField>
    <ResponseField name="total_interactions" type="number" required>
      Total number of conversation turns recorded in the database.
    </ResponseField>
    <ResponseField name="completion_percentage" type="number" required>
      Assessment completion percentage at time of export.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="full_conversation" type="object[]" required>
  Complete ordered array of conversation turns as stored in the database. Each turn contains the agent's message, the user's response, and metadata such as turn number and message type.
</ResponseField>

<ResponseField name="agent_evaluations" type="object[]" required>
  Array of per-competency evaluation records generated by the agent at assessment completion.

  <Expandable title="evaluation properties">
    <ResponseField name="competency" type="string" required>
      Competency identifier (e.g., `"INVESTIGACION_EDUCATIVA"`).
    </ResponseField>
    <ResponseField name="level" type="string" required>
      Assessed competency level: `basico`, `intermedio`, or `avanzado`.
    </ResponseField>
    <ResponseField name="confidence" type="number" required>
      Agent's confidence in the evaluation, as a float from `0.0` to `1.0`.
    </ResponseField>
    <ResponseField name="justification" type="string" required>
      Text explanation of the evaluation, including question count and average score used to determine the level.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="evaluation_summary" type="object" required>
  Aggregate statistics across all competency evaluations.

  <Expandable title="evaluation_summary properties">
    <ResponseField name="competencies_evaluated" type="number" required>
      Number of competencies with at least one recorded evaluation.
    </ResponseField>
    <ResponseField name="average_confidence" type="number" required>
      Mean confidence score across all agent evaluations. `0` if no evaluations exist.
    </ResponseField>
    <ResponseField name="evaluations_by_level" type="object" required>
      Count of evaluations at each level. Keys are level names (`basico`, `intermedio`, `avanzado`) and values are integer counts.
    </ResponseField>
  </Expandable>
</ResponseField>

### Errors

| Status | Description |
|---|---|
| `404` | Session not found in the database. |

### Example

```bash
curl --request GET \
  --url http://localhost:8000/api/admin/export/3f2504e0-4f89-11d3-9a0c-0305e82c3301
```

```json

  "export_timestamp": "2025-01-15T16:50:00.000000",

    "initial_message": "Hola, me gustaría comenzar la evaluación"
  },
  "conversation_summary": {
    "status": "completed",
    "duration_minutes": 42.3,

      "agent_message": "¡Bienvenida! Soy el asistente de evaluación...",

      "competency": "INVESTIGACION_EDUCATIVA",
```

  The `full_conversation` array may be large for long sessions. Each element corresponds to one row in the `conversation_turns` database table.
</Note>

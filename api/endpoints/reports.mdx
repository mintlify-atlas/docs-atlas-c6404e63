---
title: "Reports"
description: "Retrieve personalized assessment reports for applicants and access pre-built demo reports for UI testing."
---

Reports are generated at the end of a completed assessment. Each report contains an overall score, identified strengths, areas to improve, and a personalized learning path with recommendations for each competency.

---

## GET /api/report/{session_id}

Returns the `StudentReport` for a completed assessment session.

The report is either retrieved from cache (if the session already generated one) or generated on demand using the `ReportGenerator`. The report uses all responses collected during the `assessment` stage.

### Path parameters

<ParamField path="session_id" type="string" required>
  The UUID of the completed assessment session.
</ParamField>

### Response

<ResponseField name="assessment_id" type="string" required>
  Unique identifier for this assessment.
</ResponseField>

<ResponseField name="applicant_name" type="string" required>
  Full name of the applicant.
</ResponseField>

<ResponseField name="program_name" type="string" required>
  Name of the master's program being evaluated.
</ResponseField>

<ResponseField name="overall_score" type="number" required>
  Weighted overall score on a scale of `1.0` to `5.0`.
</ResponseField>

<ResponseField name="strengths" type="string[]" required>
  List of identified strength descriptions. Each item is a markdown-formatted paragraph highlighting a specific competency area and its relevance to the applicant's background.
</ResponseField>

<ResponseField name="areas_to_improve" type="string[]" required>
  List of improvement area descriptions. Each item is a markdown-formatted paragraph describing a competency gap and how the program addresses it.
</ResponseField>

<ResponseField name="personalized_path" type="object[]" required>
  Ordered list of personalized recommendations, one per evaluated competency.

  <Expandable title="PersonalizedRecommendation properties">
    <ResponseField name="competency_id" type="string" required>
      Identifier of the competency this recommendation applies to.
    </ResponseField>
    <ResponseField name="action" type="string" required>
      Recommended action for this competency. Examples: `"skip"`, `"take_course"`, `"take_micro_credential"`, `"review"`.
    </ResponseField>
    <ResponseField name="reason" type="string" required>
      Explanation of why this action is recommended for the applicant, referencing their score and background.
    </ResponseField>
    <ResponseField name="suggested_courses" type="string[]">
      Optional list of specific course or module names suggested for this competency.
    </ResponseField>
    <ResponseField name="estimated_time_hours" type="number | null">
      Estimated time in hours to complete the recommended action.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="estimated_completion_time" type="string" required>
  Human-readable estimate of program completion time based on the applicant's profile.
</ResponseField>

<ResponseField name="next_steps" type="string[]" required>
  Actionable next steps for the applicant to take before or at program start.
</ResponseField>

<ResponseField name="generated_at" type="string" required>
  ISO 8601 timestamp of when the report was generated.
</ResponseField>

### Errors

| Status | Description |
|---|---|
| `404` | Session not found. |
| `400` | Assessment not completed. The session exists but no responses have been recorded. |

### Example

```bash
curl --request GET \
  --url http://localhost:8000/api/report/3f2504e0-4f89-11d3-9a0c-0305e82c3301
```

<Note>
  Use `GET /api/session/{session_id}` to check `current_step` before calling this endpoint. The report is only available when `current_step` is `end` or when `report_ready` is `true` in a chat response.
</Note>

## GET /api/report/demo/{demo_user}

Returns a pre-built, realistic `StudentReport` for UI and UX testing. No active session is required.

Three demo profiles are available, each representing a distinct applicant archetype with different strengths and gaps.

### Path parameters

<ParamField path="demo_user" type="string" required>
  The demo profile to retrieve. Must be one of:

  | Value | Applicant | Profile summary |
  |---|---|---|
  | `maria` | María González Rodríguez | Experienced academic coordinator, strong in research, lower TIC pedagogy score |
  | `carlos` | Carlos Roberto Mendoza | Systems engineer, strong technical skills, limited pedagogical background |
  | `ana` | Ana Patricia Herrera | Experienced primary school teacher, strong pedagogy, developing advanced TIC skills |
</ParamField>

### Response

Same schema as `GET /api/report/{session_id}`. See the [response fields above](#response) for the full structure.

### Errors

| Status | Description |
|---|---|
| `404` | Demo user not found. The value is not one of `maria`, `carlos`, or `ana`. |

### Example

```bash
curl --request GET \
  --url http://localhost:8000/api/report/demo/ana
```

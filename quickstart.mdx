## Prerequisites

Before you begin, make sure you have the following installed:

| Requirement | Version | Notes |
|---|---|---|
| Python | 3.13+ | Required by the backend |
| Node.js | 22+ | Required by the frontend |
| uv | latest | Recommended Python package manager |
| PostgreSQL / Neon | any | A `DATABASE_URL` connection string is required |
| OpenAI API key | — | Used by the LLM agent and embedding model |

<Tip>
  [Neon](https://neon.tech) provides a serverless PostgreSQL database with a free tier that works well for local development.
</Tip>

## Set up the project

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/mafab9125/RAP_RS.git
    cd RAP_RS
    ```
  </Step>

  <Step title="Configure backend environment variables">
    Create a `.env` file in the `backend/` directory. The backend reads this file at startup via `pydantic-settings`.

    ```bash
    cp .env.example backend/.env
    ```

    Then open `backend/.env` and set the required values:

    ```env
    # Required — OpenAI is used for the LLM agent and embeddings
    OPENAI_API_KEY=sk-...

    # Required — PostgreSQL connection string (Neon or local Postgres)
    DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require

    # Optional — override defaults from config.py
    PRIMARY_LLM_MODEL=gpt-5-mini-2025-08-07
    REPORT_LLM_MODEL=gpt-5-2025-08-07
    EMBEDDING_MODEL=text-embedding-3-small
    MAX_QUESTIONS_PER_COMPETENCY=5
    PASSING_SCORE_THRESHOLD=60.0
    ASSESSMENT_TIMEOUT_MINUTES=120
    SESSION_TTL_MINUTES=60
    ```

    <Warning>
      Never commit your `.env` file. It is listed in `.gitignore` by default.
    </Warning>
  </Step>

  <Step title="Install Python dependencies">
    Run this from the `backend/` directory.

    <CodeGroup>

    ```bash uv (recommended)
    cd backend
    uv sync
    ```

    ```bash pip
    cd backend
    pip install -r requirements.txt
    ```

    </CodeGroup>

    The backend depends on FastAPI, LangGraph, LangChain, ChromaDB, SQLAlchemy, and OpenAI — all pinned in `requirements.txt`.
  </Step>

  <Step title="Install frontend dependencies">
    Run this from the `frontend/` directory.

    ```bash
    cd frontend
    npm ci
    ```
  </Step>

  <Step title="Verify the system state">
    From the `backend/` directory, run the `info` command to confirm the system can read its data files and connect to ChromaDB:

    ```bash
    python main.py info
    ```

    This prints the loaded program ID, data directory path, question bank statistics, and knowledge-base document counts — a quick sanity check before starting the API.

    <Note>
      The system loads `data/questions_optimized.json` (question bank) and optionally `MEMTIC.pdf` (knowledge base) at startup. Neither is required to start the API server — missing files produce warnings, not fatal errors.
    </Note>
  </Step>

  <Step title="Start the backend API server">
    From the `backend/` directory:

    ```bash
    python main.py api
    ```

    The API will be available at `http://localhost:8000`. The interactive Swagger UI is at `http://localhost:8000/docs`.
  </Step>

  <Step title="Start the frontend dev server">
    In a separate terminal, from the `frontend/` directory:

    ```bash
    npm run dev
    ```

    Vite starts the React app at `http://localhost:5173`.
  </Step>
</Steps>

## Verify the installation

Open `http://localhost:8000/api/health` in your browser. A healthy system returns:

```json
{
  "status": "healthy",
  "services": {
    "agent": "healthy",
    "api_key": "configured",
    "data_directory": "accessible",
    "chroma_database": "accessible"
  },
  "metrics": {
    "active_sessions": 0,
    "legacy_sessions": 0
  }
}
```

Open `http://localhost:5173` to see the React chat interface.

Open `http://localhost:8000/docs` for the full interactive API reference.

## Send your first chat message

You can start an assessment session directly from the UI, or via `curl`:

```bash
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Hola, quiero evaluar mis competencias"}'
```

The agent will respond with a greeting and ask the applicant to select a master's program. Pass the returned `session_id` in subsequent requests to maintain conversation state:

```bash
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Maestría en Ciencia de Datos", "session_id": "your-session-id"}'
```
